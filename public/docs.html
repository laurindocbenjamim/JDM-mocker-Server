Quick Start Guide

JDM MOCK is a powerful tool to create mock servers and test APIs quickly. This guide will help you understand how to use the developer console and implement the mock server in your workflow.

1. Authentication

Before interacting with the server, you need an identity.

  • Register: Use the "Create Identity" button to generate a unique User UUID.
  • Login: Select a role (Admin/Viewer) and click "Login" to obtain a JWT token.
POST /auth/register Response: { "x-user-id": "your-uuid" } POST /auth/login Body: { "role": "admin", "expiresIn": 3600000 } Response: { "token": "jwt-token", "role": "admin" }

2. Storage Explorer

Manage your data using Containers and Tables.

  • Container: A logical grouping of tables (like a database).
  • Table: Stores your data records. Creating a table requires an initial record or a schema definition.

3. API Usage

Once a table is created, you can access it via standard REST endpoints. Replace {container} and {table} with your names.

Method Endpoint Description
GET /{container}/{table} List all records
POST /{container}/{table} Create a new record
GET /{container}/{table}/:id Get a specific record
PATCH /{container}/{table}/:id Update a record
DELETE /{container}/{table}/:id Delete a record

4. Custom Endpoints

You can define custom paths for your tables to match your application's requirements. This is done during table creation or via the "Manage Columns" modal.

  • Defining a Path: In the Table Modal, look for the "Custom Endpoint" field.
  • Naming: Use paths like /v1/users or /api/orders. The server will automatically route these to your table.

Initialize Table with Custom Path

Use the _customPaths meta-field to define routes during initialization.

JavaScript
fetch('http://localhost:5000/test-db/custom-table', {
  method: 'POST',
  body: JSON.stringify({
    _init: true,
    _customPaths: {
      get: '/api/v1/users',
      post: '/api/v1/create-user'
    }
  })
});
Python
import requests
res = requests.post('http://localhost:5000/test-db/custom-table', json={
    '_init': True,
    '_customPaths': {
        'get': '/api/v1/users',
        'post': '/api/v1/create-user'
    }
})

5. Storage & Schema Management

Go beyond basic CRUD with container and schema management.

Container Operations

List all containers or delete one entirely.

JavaScript
// List Containers
fetch('http://localhost:5000/containers').then(res => res.json());

// Delete Container
fetch('http://localhost:5000/containers/my_db', { method: 'DELETE' });
cURL
curl -X GET http://localhost:5000/containers
curl -X DELETE http://localhost:5000/containers/my_db

Schema Validation

JDM MOCK supports typed validation. Use _schema to enforce types (String, Number, Boolean, Date).

JavaScript
fetch('http://localhost:5000/db/users', {
  method: 'POST',
  body: JSON.stringify({
    _schema: { name: 'String', age: 'Number', joined: 'Date' }
  })
});
Python
import requests
payload = { "_schema": { "name": "String", "age": "Number" } }
requests.post('http://localhost:5000/db/users', json=payload)

6. JWT: Header vs. Cookie Requests

JDM MOCK supports both traditional header-based and modern HttpOnly cookie-based authentication.

Header-based (Traditional API)

Send the token manually in the Authorization, CSRF-Token, or x-csrf-token header.

JavaScript
fetch('http://localhost:5000/my_db/data', {
  headers: {
    'Authorization': 'Bearer YOUR_JWT_TOKEN',
    'x-user-id': 'YOUR_UUID'
  }
});
Python
import requests
headers = {
    'Authorization': 'Bearer YOUR_JWT_TOKEN',
    'x-user-id': 'YOUR_UUID'
}
requests.get('http://localhost:5000/my_db/data', headers=headers)

Cookie-based (HttpOnly)

The server automatically sets an auth_token cookie upon login. Modern browsers will include this cookie in every subsequent request to JDM MOCK.

JavaScript (Auto-handled)
// Simply login once, and your browser handles the rest!
fetch('http://localhost:5000/auth/login', { ... }).then(() => {
   // Every fetch hereafter automatically includes the HttpOnly Cookie
   fetch('http://localhost:5000/my_db/data');
});

6. Frontend Integration

To integrate JDM MOCK into your application, use the endpoints created for each table. Below are detailed examples for all CRUD operations.

1. List All Records (GET)

Retrieve all records from a specific table.

JavaScript
fetch('http://localhost:5000/my_container/users')
  .then(response => response.json())
  .then(data => console.log(data));
Python
import requests
response = requests.get('http://localhost:5000/my_container/users')
print(response.json())
cURL
curl -X GET http://localhost:5000/my_container/users

2. Create New Record (POST)

Create a new record in the table. If the table doesn't exist, this will initialize it.

JavaScript
fetch('http://localhost:5000/my_container/users', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ name: 'Jane Doe', email: 'jane@example.com' })
}).then(res => res.json()).then(console.log);
Python
import requests
data = {'name': 'Jane Doe', 'email': 'jane@example.com'}
response = requests.post('http://localhost:5000/my_container/users', json=data)
print(response.json())
cURL
curl -X POST http://localhost:5000/my_container/users \
     -H "Content-Type: application/json" \
     -d '{"name": "Jane Doe", "email": "jane@example.com"}'

3. Update Record (PATCH)

Update specific fields of an existing record by ID or using a custom path.

JavaScript
// Standard Path
fetch('http://localhost:5000/my_container/users/RECORD_ID', {
  method: 'PATCH',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ name: 'Jane Updated' })
}).then(res => res.json()).then(console.log);

// Custom Path (e.g. /api/v1/update-user)
fetch('http://localhost:5000/api/v1/update-user/RECORD_ID', {
  method: 'PATCH',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ status: 'active' })
});
Python
import requests

# Standard Path
requests.patch(
    'http://localhost:5000/my_container/users/RECORD_ID',
    json={'name': 'Jane Updated'}
)

# Custom Path
requests.patch(
    'http://localhost:5000/api/v1/update-user/RECORD_ID',
    json={'status': 'active'}
)
cURL
# Standard Path
curl -X PATCH http://localhost:5000/my_container/users/RECORD_ID \
     -H "Content-Type: application/json" \
     -d '{"name": "Jane Updated"}'

# Custom Path
curl -X PATCH http://localhost:5000/api/v1/update-user/RECORD_ID \
     -H "Content-Type: application/json" \
     -d '{"status": "active"}'

4. Delete Record (DELETE)

Permanently remove a record from the table by ID or using a custom path.

JavaScript
// Standard Path
fetch('http://localhost:5000/my_container/users/RECORD_ID', {
  method: 'DELETE'
}).then(res => console.log('Deleted'));

// Custom Path (e.g. /api/v1/delete-user)
fetch('http://localhost:5000/api/v1/delete-user/RECORD_ID', {
  method: 'DELETE'
});
Python
import requests

# Standard Path
requests.delete('http://localhost:5000/my_container/users/RECORD_ID')

# Custom Path
requests.delete('http://localhost:5000/api/v1/delete-user/RECORD_ID')
cURL
# Standard Path
curl -X DELETE http://localhost:5000/my_container/users/RECORD_ID

# Custom Path
curl -X DELETE http://localhost:5000/api/v1/delete-user/RECORD_ID

7. Admin Console

The developer admin console (/admin.html) provides a high-level overview of all users, containers, and tables on the server. You can monitor platform stats and manage users globally.
